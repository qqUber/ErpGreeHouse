# Workflow Rule: Documentation-Driven Development

1. **Read first**: Always consult `docs/` before coding. Understand architecture, auth, integration, and MVP scope from the relevant `.md` files.
2. **Align changes**: Ensure code matches documentation. If mismatch, either fix code or propose doc update (if docs are obsolete).
3. **Keep docs updated**: After completing task, update any affected documentation to reflect new changes.
