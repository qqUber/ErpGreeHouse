# План развития системы тестирования и анализа UI CRM

## 1. Тестирование и покрытие (Testing & Coverage)

### 1.1. Автоматизированный анализ UI
- **Инструмент:** Скрипт статического анализа (`scripts/analyze_ui_components.ts`) для выявления всех интерактивных компонентов.
- **Цель:** Генерация карты покрытия тестами.
- **Метрика:** 100% критических путей покрыты E2E тестами.

### 1.2. Сценарии бизнес-логики
- **Core Flows:**
  - Создание клиента (Ручное / Telegram / QR).
  - Оформление заказа (POS).
  - Синхронизация с ERP.
- **Edge Cases:**
  - Обрыв связи (Offline mode).
  - Ошибки API (500, 401).
  - Конфликты данных.

### 1.3. User Journey Tracking
- Внедрение `logging-service` для записи действий пользователя в анонимизированном виде.
- Визуализация "воронок" прохождения сценариев в Admin UI.

## 2. Производительность и UI-Аудит (Performance)

### 2.1. Оптимизация загрузки
- **Цель:** LCP (Largest Contentful Paint) < 2.0s.
- **Методы:**
  - Code splitting (React.lazy) для тяжелых вкладок (Integrations, Settings).
  - Оптимизация бандла (Vite config).
  - Gzip/Brotli сжатие на сервере.

### 2.2. Offline-first
- Использование Service Worker (Vite PWA plugin).
- Кеширование справочников (Товары, Категории) в IndexedDB/LocalStorage.
- Очередь отложенных запросов (Background Sync) для создания чеков офлайн.

## 3. Архитектура и надежность (Reliability)

### 3.1. Кеширование и SWR
- Внедрение `TanStack Query` (React Query) для управления серверным состоянием.
- Устранение дубликатов запросов при переключении вкладок.

### 3.2. Graceful Degradation
- UI должен оставаться функциональным (read-only) при падении Backend API.
- Индикаторы "Работа офлайн" / "Синхронизация".

### 3.3. Retry Policies
- Экспоненциальная задержка (Exponential Backoff) для критических мутаций (создание чека).

## 4. Ручное управление данными (Manual Data Management)

### 4.1. Управление клиентами
- [x] Форма ручного создания клиента (ФИО, Телефон, Комментарий).
- Валидация уникальности телефона.

### 4.2. Управление товарами
- Импорт из CSV/Excel (Drag & Drop зона).
- Массовое редактирование цен.

### 4.3. Ручной ввод чеков (POS)
- [x] Выбор товаров из справочника (вместо хардкода).
- Калькулятор сдачи.
- Привязка к клиенту "на лету".

### 4.4. Аудит и RBAC
- Логирование всех ручных изменений (кто, когда, что изменил).
- Роль `Manager`: только продажи и клиенты (без настроек и товаров).

## 5. UX/UI Требования

- **Hotkeys:** `Ctrl+Enter` (Сохранить), `Esc` (Отмена), `Alt+N` (Новый клиент).
- **Feedback:** Тоасты уведомлений, прогресс-бары для долгих операций.
- **Mobile:** Адаптивная верстка таблиц (Card view на мобильных).

## 6. Критерии приемки (Acceptance Criteria)

- **Performance:** TTI < 100ms для локальных действий.
- **Reliability:** 99.9% uptime в офлайн-режиме (доступ к кешу).
- **Recovery:** Восстановление контекста (формы) после перезагрузки страницы.
