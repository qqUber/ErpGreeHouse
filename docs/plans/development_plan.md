# Комплексный План Разработки: Telegram CRM + ERPNext Loyalty Program

## 1. Обзор Проекта

### 1.1 Цель и Контекст
Создание масштабируемой CRM-системы для ритейла/кофеен с глубокой интеграцией в Telegram и ERPNext, обеспечивающей полный цикл управления лояльностью клиентов через программу бонусов.

### 1.2 Ключевые Требования
- Интеграция с ERPNext через API (без создания полноценного Frappe приложения)
- Поддержка 152-ФЗ (логирование согласий)
- Массовые рассылки до 500к пользователей
- High-load архитектура (Telegram бот)
- Программа лояльности (начисление/списание бонусов)
- Оформление заказов через бота

## 2. Архитектура Системы

### 2.1 Микросервисная Архитектура

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   Telegram      │    │   Middleware    │    │   ERPNext       │
│   (Webhooks)    │◄──►│   (FastAPI)     │◄──►│   (MariaDB)     │
└─────────────────┘    └─────────────────┘    └─────────────────┘
                                │
                                ▼
                       ┌─────────────────┐
                       │   Celery        │
                       │   Worker        │
                       └─────────────────┘
```

### 2.2 Компоненты Системы

| Компонент | Технология | Назначение | Статус |
|-----------|------------|------------|---------|
| **Middleware** | Python 3.11, FastAPI | API Gateway для Telegram, обработка вебхуков | ✅ Базовая реализация |
| **Worker** | Celery, Redis | Асинхронная обработка рассылок, очереди | ✅ Базовая реализация |
| **ERPNext Integration** | Python, REST API | Интеграция с ERPNext через Custom Fields | 🔄 В разработке |
| **ErpModern Frontend** | React 18, Vite | Веб-интерфейс менеджеров | ✅ Готово |
| **ErpModern Backend** | Python, Flask | API для фронтенда | ✅ Готово |

## 4. Данные и Структуры

### 4.1 ERPNext Custom Fields

#### Customer DocType Расширения:
```python
{
    "telegram_id": {"type": "Data", "unique": True},
    "consent_given": {"type": "Check", "default": False},
    "consent_date": {"type": "Datetime"},
    "consent_version": {"type": "Data"},
    "loyalty_balance": {"type": "Int", "default": 0}
}
```

#### Sales Order DocType Расширения:
```python
{
    "bonus_points_used": {"type": "Int", "default": 0},
    "telegram_order": {"type": "Check", "default": False},
    "telegram_client_id": {"type": "Data"}
}
```

### 4.2 Custom DocTypes

#### Loyalty Transaction:
- `telegram_client` (Link to Telegram Client)
- `transaction_type` (Select: Accrual/Redemption/Expiration/Adjustment)
- `points` (Int)
- `order_reference` (Link to Sales Order)
- `description` (Text)
- `transaction_date` (Datetime)

#### Telegram Client:
- `telegram_id` (Int, unique)
- `first_name`, `username`, `phone` (Data)
- `customer_link` (Link to Customer)
- `balance` (Int, readonly)
- `consent_given`, `consent_date`, `consent_text_version`

## 5. Этапы Разработки

### Фаза 1: Инфраструктура и Данные (Недели 1-2)
**Цель**: Подготовить инфраструктуру и структуры данных в ERPNext

#### Задачи:
1. **Скрипт инициализации ERPNext** (`init_erp_structure.py`)
   - Создание Custom Fields через API
   - Создание Custom DocTypes
   - Настройка прав доступа

2. **Клиент ERPNext** (`erp_client.py`)
   - Полная реализация REST API клиента
   - Обработка ошибок и retry логика
   - Методы для работы с Custom Fields

3. **Инфраструктура разработки**
   - Настройка окружения Docker
   - Конфигурация CI/CD пайплайнов
   - Мониторинг и логирование

**Доставка**: Работающая инфраструктура, автоматическое создание структур данных

### Фаза 2: Бизнес-логика Лояльности (Недели 3-4)
**Цель**: Реализовать основные сценарии лояльности

#### Задачи:
1. **Регистрация клиентов**
   - Сценарий `/start` в Telegram
   - Создание записей в ERPNext
   - Обработка 152-ФЗ согласий

2. **Программа лояльности**
   - Начисление бонусов за покупки
   - Списание бонусов при оплате
   - История транзакций

3. **Каталог товаров**
   - Синхронизация товаров из ERPNext
   - Категоризация для Telegram
   - Поиск и фильтрация

**Доставка**: Работающая система регистрации и базовая логика лояльности

### Фаза 3: Заказы и Оплата (Недели 5-6)
**Цель**: Полный цикл оформления заказов через Telegram

#### Задачи:
1. **Корзина и оформление**
   - Инлайн-каталог товаров
   - Управление корзиной
   - Создание Sales Order в ERPNext

2. **Оплата бонусами**
   - Расчет доступных бонусов
   - Списание при оформлении
   - Возврат при отмене

3. **Уведомления**
   - Подтверждение заказа
   - Обновления статуса
   - Напоминания

**Доставка**: Полноценная система заказов через Telegram

### Фаза 4: Рассылки и Маркетинг (Недели 7-8)
**Цель**: Инструменты массовой коммуникации

#### Задачи:
1. **Массовые рассылки**

   - Пакетная обработка (batch processing)
   - Троттлинг и защита от спама

2. **Триггерные рассылки**
   - События в ERPNext
   - Персонализированные сообщения
   - A/B тестирование

3. **Аналитика**
