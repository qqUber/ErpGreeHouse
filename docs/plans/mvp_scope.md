# MVP (Minimum Viable Product) - Omnichannel CRM + Loyalty

## 🎯 Цель MVP и Условия
Создать минимально жизнеспособный продукт (прототип/базовую версию) для централизованного управления лояльностью и рассылками во всех цифровых каналах коммуникации Кофейни. Главная бизнес-цель MVP — **экономия 50-100k ₽/мес** по сравнению с текущим решением.

**Жесткие ограничения и вводные:**
- **Срок внедрения MVP:** 3 месяца.
- **Бюджет проекта:** До 500 000 ₽.
- **Масштаб базы:** Текущая – 150 000 активных клиентов, рост до 500 000 в течение года.
- **Рекламная нагрузка:** Множество триггерных и акционных рассылок.
- **Каналы (Омниканальность):** Telegram, WhatsApp, ВКонтакте, Мобильное приложение (набор must-have функций должен быть реализован для всех каналов паритетно).
- **Суть решения:** Важно, чтобы всё стабильно работало и экономило деньги. Рассматриваются готовые open-source компоненты как база (ERPNext, Twenty CRM и др.) для укладывания в сроки и бюджет.

## 📋 Критические Функции MVP (Must-Have)

### 1. Программа Лояльности (Ядро) ✅
**Сценарий**: Клиент может получить бонусы за покупку, списать их или просто проверить баланс в любом из каналов.
**Функциональность**:
- Просмотр актуального баланса бонусов (команда `/balance` или раздел в App).
- Автоматическое начисление бонусов (например, % от чека) через API кассы или ERP.
- Списание бонусов при оплате следующего заказа (1 бонус = 1 рубль).
- Единая база лояльности для всех 4-х каналов (бесшовный опыт).

### 2. Триггерные и Акционные Рассылки ✅
**Сценарий**: Клиент получает автоматическое приглашение вернуться или массовую рассылку об акции на выходных.
**Функциональность**:
- Отправка таргетированных акционных сообщений (текст + медиа) на сегменты базы (до 500k получателей).
- Базовые триггеры: "С Днем Рождения", "Давно не заходили", "Вам начислены приветственные баллы".
- Оптимизация пропускной способности (Rate Limiting) каналов для предотвращения банов аккаунтов.

### 3. Строгий Комплаенс 152-ФЗ ✅
**Сценарий**: Пользователь начинает работу с ботом/приложением и безопасно передает свои ПДн.
**Функциональность**:
- Явный сбор согласий на обработку ПДн и отправку рекламы с кнопкой (checkbox).
- Фиксация даты, времени и версии документа (Политики конфиденциальности).
- База данных с ПДн строго находится на территории РФ.
- Возможность отзыва согласия клиентом в 1 клик с последующим удалением профиля.

## 🛠️ Технический подход MVP

### Архитектура и Инструментарий
Для соблюдения сроков (3 мес) и бюджета (500k ₽):
- **Основа системы:** Open-source CRM (ERPNext / Odoo / Twenty) или легковесный кастомный стек (FastAPI + PostgreSQL), если готовое решение усложняет процесс. На текущий момент ERPNext рассматривается как Data-Master.
- **Канальный шлюз (Омниканальность):** Единое API/Middleware, которое принимает запросы лояльности и отправляет их в Telegram-бота, WhatsApp-агрегатор, VK и App. 
- **Асинхронные очереди (Массовые рассылки):** Celery + RabbitMQ/Redis для обработки пула из 150k+ сообщений с контролем задержек API мессенджеров.

## 🎯 Что НЕ Входит в MVP (Отложено/Вырезано)
- **Создание заказов/мобильная коммерция через ботов:** Оформление корзины внутри Telegram/VK откладывается, фокус смещен на рассылки и проверку баланса лояльности.
- **Глубокая сквозная аналитика:** Сложные дашборды (PowerBI/Tableau), когортный анализ и профиль 360° — базово переносятся на следующие фазы, в MVP только статистика доставки сообщений.
- **A/B-тестирование:** Автоматическое разделение аудитории на группы для теста сообщений.
