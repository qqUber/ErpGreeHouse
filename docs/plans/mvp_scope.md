# MVP (Minimum Viable Product) - Telegram CRM + ERPNext Loyalty

## 🎯 Цель MVP

Создать работающий прототип системы для демонстрации заказчику, который показывает основные возможности интеграции Telegram с ERPNext через программу лояльности.

**Время разработки**: 2-3 недели  
**Фокус**: Ключевые функции, работающая интеграция, демонстрационные сценарии  

---

## 📋 Критические Функции MVP

### 1. Регистрация Клиентов через Telegram ✅
**Сценарий**: Пользователь заходит в бот, проходит регистрацию с телефоном и согласием 152-ФЗ

**Функциональность**:
- Команда `/start` в Telegram боте
- Сбор номера телефона
- Получение согласия 152-ФЗ через inline кнопки
- Создание клиента в ERPNext с Telegram ID
- Логирование согласия с версией и timestamp

**Демонстрация**:
```
Клиент: /start
Бот: Добро пожаловать! Для регистрации введите ваш номер телефона.
Клиент: +79991234567
Бот: Нажмите кнопку "Согласен" для подтверждения обработки данных
Клиент: [Нажимает "Согласен"]
Бот: Регистрация успешно завершена! Ваш ID: CUST-001
```

### 2. Программа Лояльности (Начисление) ✅
**Сценарий**: Клиент делает покупку, получает бонусы

**Функциональность**:
- Начисление 10% от суммы покупки в бонусах
- Создание записи о транзакции в ERPNext
- Показ баланса клиенту через `/balance`
- Минимальная сумма для начисления: 100₽

**Демонстрация**:
```
Клиент: /balance
Бот: Ваш баланс: 0 бонусов
[Админ создает заказ на 1000₽ вручную]
Клиент: /balance  
Бот: Ваш баланс: 100 бонусов
```

### 3. Использование Бонусов при Покупке ✅
**Сценарий**: Клиент оплачивает часть покупки бонусами

**Функциональность**:
- Списание бонусов при создании заказа
- 1 бонус = 1 рубль скидки
- Валидация доступного баланса
- Создание записи о списании в ERPNext

**Демонстрация**:
```
Клиент: [Создает заказ на 500₽]
Бот: У вас есть 100 бонусов. Использовать?
Клиент: [Выбирает "Использовать 50 бонусов"]
Бот: Заказ создан! Сумма к оплате: 450₽, использовано бонусов: 50
Клиент: /balance
Бот: Ваш баланс: 50 бонусов
```

### 4. Создание Заказов через Telegram ✅
**Сценарий**: Клиент оформляет заказ прямо в боте

**Функциональность**:
- Просмотр каталога товаров (минимум 3 товара)
- Добавление товаров в корзину
- Оформление заказа с выбором оплаты бонусами
- Создание Sales Order в ERPNext

**Демонстрация**:
```
Клиент: /order
Бот: Выберите товары из каталога:
[Кнопки: Кофе (200₽), Пирожное (150₽), Сэндвич (300₽)]
Клиент: [Выбирает Кофе и Пирожное]
Бот: Ваша корзина: Кофе x1 (200₽), Пирожное x1 (150₽). Итого: 350₽
Клиент: [Нажимает "Оформить заказ"]
Бот: Заказ ORD-001 создан! Итого: 350₽
```

### 5. Интеграция с ERPNext (Custom Fields) ✅
**Сценарий**: Данные синхронизируются между Telegram и ERPNext

**Функциональность**:
- Скрипт автоматического создания Custom Fields
- Telegram ID в карточке клиента
- История транзакций лояльности
- Пометка Telegram-заказов

**Демонстрация**:
```
Админ открывает клиента CUST-001 в ERPNext:
- Telegram ID: 12345
- Лояльность: 50 бонусов
- История транзакций: +100 (01.01), -50 (02.01)
- Заказы: ORD-001 (Telegram), ORD-002 (Telegram)
```

## 🎭 Демонстрационные Сценарии для Заказчика

### Сценарий 1: "Новый Клиент" (5 минут)
1. Заказчик видит пустую ERPNext
2. "Клиент" заходит в Telegram бот
3. Проходит регистрацию
4. Заказчик видит нового клиента в ERPNext с Telegram ID
5. Проверяет согласие 152-ФЗ в карточке клиента

### Сценарий 2: "Первая Покупка" (3 минуты)
1. Клиент проверяет баланс: 0 бонусов
2. Админ создает заказ на 1000₽ в ERPNext
3. Клиент проверяет баланс: 100 бонусов
4. Показываем историю транзакций в ERPNext

### Сценарий 3: "Оплата Бонусами" (4 минуты)
1. Клиент создает заказ через Telegram на 500₽
2. Выбирает оплату 50 бонусами
3. Заказ создается в ERPNext на 450₽
4. Клиент проверяет баланс: 50 бонусов
5. Показываем транзакцию списания в ERPNext

### Сценарий 4: "Админ-Интерфейс" (3 минуты)
1. Показываем список клиентов с Telegram ID
2. Открываем карточку клиента с историей лояльности
3. Показываем заказы с пометкой "Telegram"
4. Демонстрируем кнопку "Отправить сообщение в Telegram"

## 📊 Критерии Успеха MVP

### Функциональные Критерии
- ✅ Клиент может зарегистрироваться через Telegram
- ✅ Бонусы начисляются автоматически за покупки

### Демонстрационные Критерии
- ✅ Все 4 сценария работают без сбоев

## 💡 Подход к Разработке MVP

### Принципы
1. **Минимальная сложность** - работает сразу, оптимизируем позже
2. **Прямая интеграция** - без слоев абстракции, где возможно
3. **Mock данные** - если ERPNext недоступен, используем фейковые данные
4. **Живые демо** - каждая функция должна быть демонстрируема
5. **Быстрая итерация** - правки и улучшения в реальном времени

### Технический Долг
- Код без оптимизации, но рабочий
- Простая архитектура без избыточности
- Базовая обработка ошибок
- Минимальная документация
- Ручные процессы где возможно
