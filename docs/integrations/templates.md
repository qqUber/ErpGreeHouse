# Шаблоны интеграций (RF / Serbia)

## Назначение

Шаблоны — это преднастроенные профили для быстрого создания интеграций в админ‑панели (раздел «Интеграции» → поле «Шаблон»).

## Доступные шаблоны

### RU

1) **Касса РФ: входящий webhook чека** (`pos_webhook`)
- Используется, когда кассовое ПО/оборудование умеет отправлять HTTP‑уведомления о закрытом чеке.
- Создаёт транзакцию CRM и привязывает её к клиенту.
- Payload и требования: см. `pos_webhook.md`.

2) **События: webhook в внешнюю систему** (`outbound_webhook`)
- Используется для отправки событий во внешние системы (в т.ч. коннекторы мессенджеров).
- Конфигурация и формат: см. `outbound_webhook.md`.

### RS

1) **Касса Сербия: входящий webhook фискального чека** (`pos_webhook`)
- Базовый сценарий, аналогичный RU, но рассчитан на источники, которые уже прошли фискализацию в локальном контуре.
- В дальнейшем может быть расширен под специфику eFiskalizacija (ESIR/LPFR) через отдельные драйверы.

2) **События: webhook в внешнюю систему (Serbia)** (`outbound_webhook`)
- Аналогично RU — доставка событий CRM во внешние контуры.

## API

- Список шаблонов: `GET /api/v1/integrations/templates` (требует `x-admin-secret`)
