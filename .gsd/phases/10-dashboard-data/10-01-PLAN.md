# Plan 10-01: Enhanced Dashboard API Endpoints

## Objective
Create comprehensive backend API endpoints to support an enterprise-grade dashboard with real-time operational data, marketing analytics, and customer insights from all database tables.

## Context
The current dashboard only shows basic KPIs and a revenue trend. We're missing:
1. Operational data (hourly breakdowns, staff activity)
2. Marketing events (triggers, campaigns)
3. Customer insights (segments, loyalty tiers)
4. Product analytics (top sellers, inventory)
5. Integration health status

## Tasks

### Task 1: Create Operational Data Endpoint
**Files**: `middleware/app/dashboard_api.py` (new file)

Create new API router with endpoints:

1. **GET /api/v1/dashboard/operational**
   - Today's hourly transaction counts and revenue
   - Current active staff/operators
   - Average transaction time
   - Peak hours identification
   - Top products today (with quantities)

2. **GET /api/v1/dashboard/performance**
   - Staff performance metrics
   - Transactions per operator
   - Average check size per operator
   - Customer satisfaction indicators

### Task 2: Create Marketing Dashboard Endpoint
**Files**: `middleware/app/dashboard_api.py`

Create endpoint:

1. **GET /api/v1/dashboard/marketing**
   - Active campaigns count
   - Recent trigger executions (last 24h)
   - Segment distribution (pie chart data)
   - Campaign performance (open rates, click rates)
   - Upcoming scheduled campaigns

### Task 3: Create Customer Insights Endpoint
**Files**: `middleware/app/dashboard_api.py`

Create endpoint:

1. **GET /api/v1/dashboard/customers**
   - New customers today/this week/this month
   - Customer segments breakdown
   - Loyalty tier distribution
   - Top customers by spend
   - Customers with birthdays this week

### Task 4: Create Product Analytics Endpoint
**Files**: `middleware/app/dashboard_api.py`

Create endpoint:

1. **GET /api/v1/dashboard/products**
   - Top 10 selling products today
   - Products by category performance
   - Low stock alerts (if inventory tracking)
   - Product return rates

### Task 5: Create Integration Health Endpoint
**Files**: `middleware/app/dashboard_api.py`

Create endpoint:

1. **GET /api/v1/dashboard/integrations**
   - Integration status (Telegram, VK, ERP)
   - Recent webhook deliveries (success/failure)
   - Pending deliveries queue
   - Last sync timestamps

### Task 6: Wire Up Routes
**Files**: `middleware/app/main.py`

Import and include the new dashboard router:
```python
from app.dashboard_api import router as dashboard_router
app.include_router(dashboard_router, prefix="/api/v1")
```

## Verification

Test each endpoint:
```bash
curl http://localhost:8000/api/v1/dashboard/operational
curl http://localhost:8000/api/v1/dashboard/marketing
curl http://localhost:8000/api/v1/dashboard/customers
curl http://localhost:8000/api/v1/dashboard/products
curl http://localhost:8000/api/v1/dashboard/integrations
```

All should return JSON with real data from seeded database.

## Success Criteria
- [ ] All 5 API endpoints return 200 with valid JSON
- [ ] Data comes from real database queries
- [ ] No hardcoded/mock data
- [ ] Proper error handling
