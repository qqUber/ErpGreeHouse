<objective>
Conduct a comprehensive security audit of the ErpGreeHouse system to identify, document, and prioritize security vulnerabilities. The audit will cover both frontend (React/Vite) and backend (FastAPI) components, focusing on authentication, authorization, data protection, and compliance with security best practices.

Purpose: To ensure the system is secure against common vulnerabilities before the v1.1 Security milestone is completed.
Output: A detailed security audit report with vulnerability prioritization and remediation plan.
</objective>

<context>
@.gsd/STATE.md
@.gsd/ROADMAP.md
@admin-ui/src/api.ts
@admin-ui/src/stores/auth.tsx
@middleware/app/auth.py
@middleware/app/admin_auth_api.py
@middleware/app/security.py
@middleware/.env
</context>

<must_haves>

- Full security audit of authentication and authorization systems
- Vulnerability scanning of API endpoints
- Analysis of data protection and encryption practices
- Review of compliance with GDPR/CCPA requirements
- Documentation of all identified vulnerabilities with severity ratings
- Prioritization plan for addressing security issues
- Remediation recommendations for each vulnerability

</must_haves>

<should_haves>

- Automated security scanning using OWASP ZAP or similar tools
- Penetration testing of critical endpoints
- Review of third-party dependencies for known vulnerabilities
- Security audit of frontend code (XSS, CSRF, etc.)
- Analysis of server configuration and deployment practices

</should_haves>

<verification>

Run comprehensive security scans and review all audit results:

1. Bandit scan on backend: `bandit -r middleware/app -f html -o audit/bandit-report.html`
2. pip-audit: `pip-audit -r middleware/requirements.txt -f html > audit/pip-audit-report.html`
3. npm audit: `cd admin-ui && npm audit --audit-level high`
4. eslint security scan: `cd admin-ui && npx eslint src --plugin security --format html --output-file audit/eslint-security-report.html`

Review all generated reports for vulnerabilities.
</verification>
