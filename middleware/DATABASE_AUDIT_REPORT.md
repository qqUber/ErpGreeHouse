## 3. PostgreSQL Compatibility Issues

### 3.1 Critical Issues

| Issue | Description | Impact |
|-------|-------------|--------|
| `datetime('now')` in DEFAULT | SQLite function not available in PostgreSQL | Migration required |
| `INTEGER PRIMARY KEY` | Should be `BIGSERIAL` in PostgreSQL | Auto-increment change needed |
| JSON stored as TEXT | Should use `JSONB` for performance | Query changes needed |
| No case-insensitive search | SQLite LIKE works for ASCII only | Need PostgreSQL `ILIKE` |

### 3.2 Recommended PostgreSQL Migrations

```sql
-- 1. Convert INTEGER PRIMARY KEY to BIGSERIAL
ALTER TABLE customers ALTER COLUMN id ADD GENERATED BY DEFAULT AS IDENTITY;

-- 2. Replace datetime() with CURRENT_TIMESTAMP
ALTER TABLE customers ALTER COLUMN created_at SET DEFAULT CURRENT_TIMESTAMP;

-- 3. Convert TEXT JSON to JSONB
ALTER TABLE customers ALTER COLUMN preferences_json TYPE JSONB USING preferences_json::jsonb;
```

### 6.2 Cyrillic & Serbian Latin Support

**Finding: LIMITED SUPPORT**

SQLite's `LIKE` operator:
- ✓ Case-insensitive for **ASCII characters only** (a-z, A-Z)
- ✗ **NOT** case-insensitive for:
  - Russian Cyrillic (а-я, А-Я)
  - Serbian Latin diacritics (ć, č, đ, š, ž)

**Example issues:**
| Search Term | Stored Name | Found? |
|-------------|-------------|---------|
| `%John%` | John Smith | ✓ Yes |
| `%JOHN%` | John Smith | ✓ Yes |
| `%Иван%` | Иван Петров | ✓ Yes (exact case) |
| `%иван%` | Иван Петров | ✗ No (SQLite default) |
| `%Jovanović%` | Nikola Jovanović | ✓ Yes |
| `%jovanović%` | Nikola Jovanović | ✗ Probably No |

### 6.3 Test Created

Created [`middleware/tests/unit/test_search_cyrillic_serbian.py`](middleware/tests/unit/test_search_cyrillic_serbian.py) to verify search behavior.

### 6.4 Recommendations for Better Search

1. **For PostgreSQL migration**: Use `ILIKE` instead of `LIKE`
2. **For better Unicode support**: Use case-insensitive collation
3. **For advanced search**: Consider full-text search with:
   ```sql
   SELECT * FROM customers 
   WHERE to_tsvector('russian', full_name) @@ plainto_tsquery('russian', 'иван')
   ```
